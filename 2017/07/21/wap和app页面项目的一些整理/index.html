wap和app页面项目的一些整理

  1. 分平台
    • app
    • wap
  2. 分设备
    • ios
    • andriod

多数情况，在不同的平台下需要显示不同的内容

  1. 一般在wap下需要显示财新的头部导航栏和底部的版权信息栏。 app则无需头尾。
  2. 在不同的平台下有不同的需求设置，如在app下微信分享的内容和wap端微信分享的内容设置方式是不同的。
  3. 一些组建插件在不同的平台和不同的设备上表现存在差异，可能需要做响应的调整。

建议在应用中设置一份配置文件 设置相关常量

exp:

const platform = ...   // 平台类型
const device = ...     // 设备类型
const deviceWidth = ... // 设备宽度
...

信息梳理

比较繁琐的一部分是分享的页面信息，还需要经常修改

  1. 页面标题 title
  2. 页面 keywords
  3. 页面 description
  4. 微信分享到朋友的标题
  5. 微信分享到朋友圈的标题
  6. 分享内容等缩略图

最好能事先根据相关页面收集整理出以上信息做成Excel，然后相关微信分享信息可以参考上篇微信分享，如果手头没有类似信息可以找产品

夜间模式和白天模式，这个应该是属于应用一个状态。相关信息存储在cookie中，键名isNightMode，

  • isNightMode = 1夜间模式
  • isNightMode = 0 白天模式

夜间模式的相关设置

应用启动时，可以从cookie里读取相关cookie并判断其值再做设置。简单来说可以在html里最外面一层设置一个class和id。简单可以用一个night.css来覆盖白天模式的样式。精细一点的可以在每个组件里就写好夜间白天模式的相关样式。
当我们的页面嵌入在**app里时当app设置完日夜间模式时，我们的页面需要刷新一下才会显示成日夜间模式。解决这个问题可以在html页面里调用app的相关方法

exp: 这段代码也有些不同，我们的页面作为iframe嵌入在他们的页面里。app设置完日夜间模式后默认会调用setNightMode的方法，在我们的应用场景下，这段代码还是写在他们页面里，所以就只能刷新iframe来实现了
   var ifr = document.getElementById("iframe") // 获取页面iframe 
    function setNightMode(mode){
      if(mode == "night"){
       document.body.className = "night";
       var nightUrl = ifr.src;
       nightUrl = nightUrl.replace(/&isNight=1|\?isNight=1|&isNight=0|\?isNight=0/g,"");
       if(nightUrl.indexOf("?") != -1){
        ifr.src = nightUrl + "&isNight=1";
       } else {
        ifr.src = nightUrl + "?isNight=1";
       }
       isNightMode = "1"
      } else {
       document.body.className = "";
       var dayUrl = ifr.src;
       dayUrl = dayUrl.replace(/&isNight=1|\?isNight=1|&isNight=0|\?isNight=0/g,"");
       ifr.src = dayUrl
       isNightMode = "0"
      }
     }
